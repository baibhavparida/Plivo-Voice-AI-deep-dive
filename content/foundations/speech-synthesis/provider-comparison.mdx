## Cartesia (Sonic)

### Company Background

Founded by researchers who developed State Space Models (S4, Mamba).

### Architecture Approach

State Space Models (SSMs) provide the core innovation:

```
SSM: dx/dt = Ax + Bu
     y = Cx + Du
```

**Advantages of SSMs**:
- Sub-quadratic scaling (vs. quadratic for Transformers)
- Constant-time processing for new inputs
- Native streaming capability
- Efficient for long sequences

### Sonic Model Specifications

- TTFA: ~40-90ms (fastest in market)
- Architecture: ~10:1 SSM:attention ratio
- Streaming: True word-by-word synthesis

### Sonic 3 Features

- Real-time laughter and emotion
- ~190ms end-to-end latency
- Internal testing achieved 25ms synthesis

### Pricing (2025)

- Free tier available
- ~$0.05 per 1,000 characters
- Approximately 1/5 the cost of ElevenLabs

<Callout type="success" title="Strengths">
Lowest latency, efficient architecture, competitive pricing
</Callout>

<Callout type="warning" title="Considerations">
Smaller voice library, newer in market
</Callout>

## OpenAI TTS

### Architecture Approach

Two architectural options:

**1. Chained Architecture**:
```
gpt-4o-transcribe -> gpt-4.1 -> gpt-4o-mini-tts
```

**2. Multimodal S2S** (Realtime API):
```
Audio -> gpt-4o-realtime-preview -> Audio
```

### Models

| Model | Description | Use Case |
|-------|-------------|----------|
| tts-1 | Standard TTS | High-volume applications |
| tts-1-hd | High-definition | Quality-critical |
| gpt-4o-mini-tts | Latest, steerable | Instructable voice style |

### gpt-4o-mini-tts Features

- Instruction-following: Control not just what but how to speak
- 35% lower WER on speech benchmarks
- 11 built-in voices
- Custom voice support (limited access)

### Pricing (2025)

| Model | Price per 1M characters |
|-------|------------------------|
| tts-1 | $15 |
| tts-1-hd | $30 |
| gpt-4o-mini-tts | $0.60 per 1M input characters |

<Callout type="success" title="Strengths">
Cost-effective, integrated with GPT ecosystem, steerable
</Callout>

<Callout type="warning" title="Considerations">
Limited voice cloning (requires sales contact), higher latency for basic API
</Callout>

## Amazon Polly

### Architecture Approach

Three engine types:

| Engine | Technology | Description |
|--------|------------|-------------|
| Standard | Concatenative | Lower cost, basic quality |
| Neural (NTTS) | Seq2seq + neural vocoder | Natural prosody |
| Generative | Billion-parameter transformer | Most human-like |

### Generative Engine (2024)

- Billion-parameter transformer for text-to-speech-codes
- Convolution-based decoder for waveform generation
- Streamable, incremental synthesis

### Voice Options

- Standard voices: Lower cost, basic quality
- Neural voices: 24kHz, natural prosody
- Long-form voices: Optimized for extended content
- Generative voices: Most human-like, emotionally adaptive

### SSML Support

Full SSML with Polly-specific extensions.

### Pricing (2025)

| Engine | Price per 1M characters |
|--------|------------------------|
| Standard | $4.00 |
| Neural | $16.00 |
| Generative | $30.00 |
| Free tier | 5M standard / 1M neural (first year) |

<Callout type="success" title="Strengths">
AWS integration, SSML support, multiple engine options
</Callout>

<Callout type="warning" title="Considerations">
Voice cloning limited, fewer voice options than some competitors
</Callout>

## Azure Neural TTS

### Architecture Approach

Three-component neural architecture:
1. Text Analyzer -> Phoneme sequence
2. Neural Acoustic Model -> Acoustic features
3. Neural Vocoder -> Waveform

### Key Features

- 400+ voices in 140+ languages
- Multilingual single voice (speak multiple languages)
- 24kHz standard, 48kHz HD options
- Custom Neural Voice support

### Custom Neural Voice

- Requires voice talent consent verification
- Speech Studio for training and deployment

### HD Voices (2024)

### SSML Support

## Open-Source Solutions

### Coqui TTS / XTTS-v2

**Status**: Company closed January 2024; open-source continues.

**XTTS-v2 Capabilities**:
- Zero-shot voice cloning with 6 seconds of audio
- 17 languages supported
- Cross-language cloning
- Style/emotion transfer

**Architecture**: End-to-end model combining:
- GPT-style autoregressive prior
- HiFi-GAN vocoder
- Speaker conditioning from reference audio

**Performance**:
- Streaming latency: < 150ms (GPU)
- Quality: Comparable to commercial offerings

**Licensing**: Coqui Public Model License (non-commercial)

**GitHub**: github.com/coqui-ai/TTS

### Bark (Suno AI)

**Architecture**: GPT-style, three-stage transformer pipeline

```
Text -> Semantic Tokens -> Coarse Acoustic Tokens -> Fine Acoustic Tokens -> Audio
```

**Unique Features**:
- Generates non-speech sounds (laughter, sighing, music)
- 100+ speaker presets
- Multilingual

**Limitations**:
- No voice cloning support
- Output limited to ~13-14 seconds
- Stochastic (may diverge from prompt)

**Requirements**: ~12GB GPU memory (full), ~8GB (small version)

**License**: MIT (commercial use allowed)

### Tortoise TTS

**Architecture**: Five neural networks
1. Autoregressive decoder (GPT-2 style)
2. Diffusion model
3. CLVP (Contrastive Language-Voice Pretraining)
4. UnivNet vocoder
5. Conditioning encoder

**Quality Focus**: Emphasizes naturalness over speed.

**Performance**:
- Very slow: ~2 minutes per sentence on K80
- Optimized: 0.25-0.3 RTF with streaming on 4GB VRAM

**Voice Cloning**: Multiple reference clips supported.

**License**: Apache 2.0

## Selection Guide

### By Use Case

| Use Case | Recommended Provider | Rationale |
|----------|---------------------|-----------|
| Real-time voice agents | Cartesia, Deepgram | Lowest latency |
| Highest quality content | ElevenLabs | Best expressiveness |
| Enterprise deployment | Azure, Deepgram | Security, compliance |
| AWS ecosystem | Amazon Polly | Native integration |
| GCP ecosystem | Google Cloud TTS | Native integration |
| Budget-conscious | OpenAI, Polly Standard | Lowest cost |
| Voice cloning | ElevenLabs, Coqui | Best cloning features |
| Self-hosted | Coqui XTTS, Bark | No API costs |

### By Latency Requirements

| Requirement | Provider Options |
|-------------|-----------------|
| < 100ms TTFA | Cartesia Sonic |
| < 200ms TTFA | Deepgram Aura, ElevenLabs Flash |
| < 300ms TTFA | Most providers |
| Latency not critical | Any provider |

### By Language Needs

| Requirement | Best Options |
|-------------|--------------|
| English only | Deepgram, Cartesia |
| European languages | ElevenLabs, Google, Azure |
| Asian languages | Azure, Google |
| 100+ languages | PlayHT, Azure |
